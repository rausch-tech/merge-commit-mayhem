# 05 – AI Asset Pack Master Prompt

Dieses Dokument enthält Master-Prompts für die Generierung eines konsistenten AI-Asset-Packs.

Wichtig: Nicht alles in einem Bild generieren. Besser sind einzelne Asset-Familien:

1. Logo
2. UI-Icons
3. Action-Icons
4. Sabotage-Icons
5. Panels
6. Buttons
7. Room Labels
8. Role Badges
9. Environment Props
10. Characters

---

# 1. Master Prompt

```text
Create a complete modular 2D game asset pack for a multiplayer office-chaos game called “Merge Conflict Mayhem: Lunch Break Edition”.

The asset pack is for a Godot-based game MVP and should include:
- logo
- UI icons
- action icons
- sabotage icons
- panel backgrounds
- buttons
- room labels
- badges / role markers

Overall art direction:
A clean, stylized, modern 2D game UI aesthetic with a slightly isometric / indie-game feel, inspired by humorous tech culture, software development, DevOps chaos, office satire, and multiplayer party games.
The style should feel:
- playful but competent
- polished but not overly complex
- humorous but readable
- game-ready and production-friendly
- suitable for a Godot UI and a top-down / isometric office sabotage game

Visual style:
- 2D digital game art
- crisp edges
- slightly cartoonish
- subtle shading
- soft ambient gradients if needed
- high readability at small sizes
- clean silhouettes
- not photorealistic
- not painterly
- not messy
- no excessive texture noise
- no overly realistic 3D look
- no flat corporate icon style only; it should still feel like game UI

Theme and tone:
A dev-team emergency during a failing software release.
Think:
- merge conflicts
- build failures
- incident response
- coffee dependency
- legacy systems
- suspicious saboteurs
- meetings as hazards
- office/server-room humor

Color system:
Use a consistent UI/game color palette:
- dark navy / charcoal for base panels and backgrounds
- bright orange for title accents and coffee-related utility
- green for tasks / success / progress
- blue for infrastructure / stability / technical actions
- red for incidents / sabotage / warnings
- purple for meetings / social actions / pull requests
- yellow for emphasis / alert / legacy weirdness
The overall look should work on dark UI backgrounds.

Output requirements:
- create assets as isolated individual elements, not as one poster or one mockup
- transparent background wherever possible
- centered composition
- clean cutout shapes
- no mockup scene
- no perspective distortion unless requested
- no embedded UI screenshots
- no unnecessary text except where explicitly requested
- no watermark
- no decorative background scene

Rendering requirements:
- game-ready asset style
- readable at small and medium sizes
- suitable for export as PNG
- consistent across all asset groups
- visual consistency is very important

Asset categories to generate:
1. Main game logo
2. Task and UI icons
3. Sabotage icons
4. Action/ability icons
5. UI panels and frames
6. Buttons
7. Room labels
8. Role badges / role icons

Each asset should look like part of one coherent game universe.

Important:
Do not generate one combined sheet unless explicitly requested.
Generate isolated, individual assets or tightly grouped icon sets with consistent spacing.
Prefer transparent backgrounds.
Avoid text-heavy designs except for the logo and room labels.
```

# 2. Logo Prompt

```text
Create a polished main game logo for a multiplayer office-chaos game called:

“Merge Conflict Mayhem”

Style:
A bold, high-energy indie game logo for a humorous tech sabotage game.
The logo should feel like a real game title, combining software engineering chaos, office humor, and multiplayer fun.

- large, bold title typography
- custom stylized lettering
- dynamic composition
- readable and iconic
- slightly comic / game-like
- polished, professional, fun
- not childish
- not generic corporate branding
- not photorealistic
- no mockup scene

- “MERGE” should feel strong, technical, and structural
- “CONFLICT” should feel chaotic, cracked, or electrically unstable
- “MAYHEM” should feel explosive, energetic, and playful

Possible visual motifs:
- small code brackets </> or branch/merge motifs
- subtle glitch / lightning / warning accents
- minor coffee or office-chaos references
- tiny sparks, incident accents, or dev-themed details
- but keep the logo clean and usable

Color palette:
- white / light gray for structure
- orange / warm amber for energy accents
- red for chaos emphasis
- dark outlines for readability
- small blue or green tech accents if useful

Output requirements:
- transparent background
- centered composition
- clean cutout
- no scene
- no extra text beyond the game title and subtitle
- game-ready
- suitable for title screen, splash screen, and UI header
```
