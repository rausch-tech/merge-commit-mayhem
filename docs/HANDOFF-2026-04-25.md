## Was funktioniert (Spielfluss)

1. **Lobby:** Drei Tabs joinen mit Raumcode `ABCD`. Erster ist Host, sieht „Runde starten".
2. **Start:** Host klickt → Rollen werden privat verteilt (1× Vibe Coder, Rest Developer). HUD zeigt eigene Rolle (Chaos rot hinterlegt).
3. **Movement:** WASD oder Pfeiltasten. Wenn Coffee=0 oder Mandatory Meeting aktiv → halbierte Speed (60 px/s).
4. **Tasks:** Vier Marker auf der Canvas (Open Space, Server Room, Kitchen). In der Sidebar links steht die Liste.
   - In den 40-px-Radius laufen → Task bekommt einen gestrichelten gelben Ring
   - `E` (oder Space) drücken und halten → Progress-Ring wird gelb
   - Bei `required_seconds` → Reward (release/pipeline/coffee), 8s Cooldown
   - Switch-Sound spielt bei Completion
5. **Sabotage (Chaos):** Drei Buttons unten rechts. Klick → Effekt + Cooldown.
   - CI/CD Rot: Pipeline -20
   - Kaffee leer: Coffee = 0 (alle langsam)
   - Mandatory Meeting: Alle 5 s langsam
6. **Win:** Pipeline ≤ 0 (chaos), Release ≥ 100 (release), Timer 0 (chaos). Endscreen-Overlay mit Banner + Reason + Rollen-Reveal + Stats.
7. **Neue Runde:** Host klickt „Zurück zur Lobby" → Reset → Lobby-Screen mit gleicher Spielerliste, neue Rollen-Zuweisung beim nächsten Start.

## Kritische Designnotizen (Sven 2026-04-25)

Siehe `merge_conflict_mayhem_project/docs/10_sven_notes_2026-04-25.md`.

**1. Map muss scrollbar werden — KRITISCH für Social Deduction.**
Aktuell sieht jeder die ganze Karte und damit auch wer sabotiert. Das zerstört das Gameplay. Lösung: Kamera-folgende-Spieler im Frontend, Map deutlich größer (z. B. 2400×1600). Backend bleibt unverändert. Sollte direkt nach diesem Slice angegangen werden.

**2. „Außer Gefecht setzen" (kein „Kill"-Wording!) — Roadmap.**
Saboteur soll Spieler aus dem Team nehmen können, mit witzigem Tech-Gadget-Naming. Kommt nach Map-Scrolling.

## Bekannte Schwächen / Polish-Punkte

- **Task-Marker als 2-Buchstaben-Kreise** (Platzhalter, bis Spritesheets gesliced sind)
- **Sabotage-Buttons reine CSS-Buttons** (keine echten Icons)
- **Logo-Skalierung:** logo.png ist groß (2 MB) und unsharp gerendert — vermutlich braucht es eine kleinere PNG-Variante oder SVG
- **HUD wird etwas eng bei kleineren Bildschirmen** — kein Responsive-Layout für < 800 px Breite
- **Task-Cooldown-Indikator in der Sidebar zählt in Sekunden hoch** statt einer schönen Progressbar
- **Audio-Volume hard-coded auf 0.3** — keine Mute-Option, kein Slider
- **Kein Error-Toast wenn `trigger_sabotage` während Cooldown** (Server schickt zwar `error`, aber Client zeigt's nur als HUD-Banner-Flash 4 s)

Alles davon ist Polish, nichts blocking.
